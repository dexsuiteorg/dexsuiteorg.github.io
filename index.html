Abstract

Abstract— DexSuite is a dexterity-first simulation stack that unifies manipulators, end-effectors, and task definitions under a single Genesis-based interface. The framework normalizes contact parameters, workspace sampling, and controller APIs so that parallel grippers, multi-finger hands, and bimanual setups can be swapped without rewriting task code. It integrates photoreal rendering, GPU-vectorized rollouts, and curated teleoperation datasets collected with Manus gloves and Vive trackers.

We provide ready-to-use benchmarks that span rigid, articulated, and deformable objects across single-arm and two-arm variants. Each benchmark includes reproducible presets, calibrated object assets, and multi-modal observations, enabling meaningful comparisons between imitation learning and reinforcement learning policies on dexterous manipulation tasks.


Dexsuite Framework

Every environment in DexSuite inherits from a lightweight Base Environment that spins up Genesis, loads robots, configures cameras, and manages batching. A Task class then defines the scene: object assets, randomization, reward shaping, and termination logic. Because the infrastructure layer stays constant, swapping between manipulators, grippers, or even single-arm versus bimanual layouts only requires changing the task binding.


Data Collection

DexSuite ships with native teleoperation tooling so we can record expert demonstrations directly inside the framework. Vive trackers provide arm pose, Manus gloves stream finger angles, and everything maps into Genesis through our ROS2 bridge. Each environment includes curated datasets collected with this setup, which is how we reach 150k+ glove-conditioned frames across single-arm and bimanual variants.

The same tooling is exposed to users: plug in the supported hardware, select the robot-hand pairing, and the framework handles calibration, workspace alignment, and logging so additional demonstrations can be captured without writing glue code.
